# 饭小二 Demo 演示脚本

> 演示时长：~8 分钟 | 打开浏览器 `http://localhost:8084`

---

```bash
python preflight.py
```

## 标准演示流程（6步）

### 1. 开场（30秒）

**操作**：打开 `http://localhost:8084`，等待右上角绿点出现

**话术**：
> "这是饭小二，我们基于 LangGraph 多智能体框架搭建的餐饮AI客服。
>  顶部绿灯表示后端就绪，下面我们来看几个场景。"

---

### 2. 确定性查询（1分钟）

**操作**：依次点击快捷按钮

| 按钮 | 演示点 |
|------|--------|
| 全部菜单 | 结构化输出，无需LLM |
| 营业时间 | FAQ精确回答 |
| 优惠活动 | 活动信息查询 |

**话术**：
> "这些都是确定性查询，直接走规则引擎，响应在毫秒级。
>  不消耗token，不调用大模型。"

---

### 3. RAG 推荐（1.5分钟）

**操作**：点击「老人推荐」

**话术**：
> "现在用户问'推荐老人吃的菜'。
>  Router先识别为'菜品推荐'意图，然后RecommendAgent调用RAG检索知识库——
>  我们把菜单和FAQ向量化存储在ChromaDB里，检索出相关菜品后组装成推荐。"

**操作**：点击「花生过敏」

**话术**：
> "再看一个忌口场景。系统会识别过敏原，过滤掉含花生的菜品，只推荐安全的。"

---

### 4. 预算场景 + 多轮（1.5分钟）

**操作**：点击「3人150元」

**话术**：
> "预算推荐场景。用户说3个人150元预算，系统计算人均50元，
>  从菜单里筛选符合预算的组合推荐。"

**操作**：继续输入「改成两个人」

**话术**：
> "注意这里没有重新说明预算，系统自动继承上轮的预算信息，
>  重新计算人均后给出新的推荐。这是多轮对话状态管理。"

---

### 5. 投诉处理（1分钟）

**操作**：点击「投诉」快捷按钮

**话术**：
> "投诉场景。我们预置了8类常见投诉的快速响应。
>  实际生产环境会对接工单系统。这个走确定性规则，不调LLM。"

---

### 6. 边界 + 新会话（1分钟）

**操作**：手动输入「那个多少钱」

**话术**：
> "模糊提问——'那个'是什么？系统识别出缺少具体菜品名称，
>  不会瞎猜，而是友好地请用户明确。"

**操作**：点击「New Session」按钮，再输入一个问候

**话术**：
> "点击New Session清空会话状态，开始全新对话。"

## 关键技术点总结（30秒）

| 技术 | 说明 |
|------|------|
| LangGraph | 多智能体编排，条件路由 |
| 三层架构 | 规则 → 结构化推荐 → LLM+RAG |
| ChromaDB RAG | 向量检索 + 结构化数据混合 |
| 确定性优先 | 30+场景不调LLM，毫秒响应 |
| 多轮状态 | 上下文继承，follow-up检测 |

## 常见追问 Q&A

**Q: 为什么不用纯LLM？**
> 成本和延迟。纯LLM每次2-6秒，我们90%的查询走规则引擎，<100ms。

**Q: 扩展新菜品怎么办？**
> 更新 `menu.json`，重启服务自动重建向量库。无需改代码。

**Q: 支持多语言吗？**
> 当前中文。Router支持关键词扩展，RAG可加入多语言文档。

**Q: 生产部署方案？**
> Docker + Nginx反代，具体见 `docs/deployment_guide.md`。
