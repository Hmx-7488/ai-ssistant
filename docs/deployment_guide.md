# 扣子平台部署指南

## 一、准备工作

### 1.1 注册扣子账号
1. 访问扣子官网：https://www.coze.cn
2. 点击"注册"，使用手机号或邮箱注册
3. 完成实名认证（可选，但建议完成）

### 1.2 了解扣子平台
扣子平台主要功能：
- **Bot** - 创建AI机器人
- **工作流** - 可视化编排AI处理流程
- **插件** - 扩展AI能力
- **知识库** - 上传文档供AI检索

---

## 二、创建Bot

### 2.1 新建Bot
1. 登录扣子平台
2. 点击"创建Bot"
3. 填写基本信息：
   - **Bot名称**：饭小二餐饮客服
   - **Bot描述**：幽默、专业、快速响应的餐饮AI客服
   - **图标**：上传餐饮相关图标

### 2.2 配置Bot人设
在"人设与回复逻辑"中，粘贴以下提示词：

```
# 角色设定
你是"饭小二"，一家网红餐厅的AI客服，人称"美食界相声演员"。

# 核心原则
1. **幽默但不低俗** - 让顾客笑着点完餐
2. **专业但不刻板** - 用大白话讲专业知识
3. **热情但不油腻** - 像朋友推荐而非销售推销
4. **快速但不敷衍** - 3秒内响应，内容不打折

# 语言风格
- 适度使用网络热梗
- 善用比喻和拟人化描述菜品
- 语气亲切，像邻家小哥/小姐姐
- 关键信息清晰准确，幽默不油腻

# 回复格式
- 简短问题：1-2句话，带个表情或梗
- 推荐菜品：菜品名+一句话亮点+价格
- 复杂咨询：分点说明，清晰易读
```

## 三、创建工作流

### 3.1 进入工作流编辑器
1. 在Bot配置页面，点击"工作流"
2. 点击"新建工作流"
3. 命名为"餐饮客服主流程"

### 3.2 添加节点

#### 步骤1：添加开始节点
- 工作流会自动创建开始节点
- 该节点接收用户输入

#### 步骤2：添加LLM节点（意图识别）
1. 点击"+"添加节点
2. 选择"LLM"节点
3. 配置：
   - **节点名称**：意图识别
   - **模型**：豆包大模型（响应快）
   - **提示词**：
   ```
   你是餐饮客服意图识别专家。快速判断用户意图。
   
   用户输入: {{input}}
   
   请从以下类别中选择一个：
   - 点餐咨询
   - 菜品推荐
   - 价格查询
   - 订座服务
   - 投诉建议
   - 订单查询
   - 闲聊互动
   - 其他咨询
   
   只输出类别名称。
   ```

#### 步骤3：添加条件分支节点
1. 添加"条件"节点
2. 配置分支条件：
   - 条件1：意图 == "点餐咨询" → 点餐服务
   - 条件2：意图 == "菜品推荐" → 推荐服务
   - 条件3：意图 == "投诉建议" → 投诉处理
   - 默认：通用对话

#### 步骤4：添加知识库节点
1. 添加"知识库"节点
2. 选择已上传的知识库
3. 配置检索参数：
   - 返回数量：3
   - 相似度阈值：0.7

#### 步骤5：添加各场景LLM节点
为每个场景添加LLM节点，参考 `workflows/main_workflow.yaml` 中的配置。

#### 步骤6：添加结束节点
- 连接所有场景节点到结束节点
- 输出最终回复

### 3.3 连接节点
按照 `workflows/main_workflow.yaml` 中的 edges 配置连接节点。

## 四、上传知识库

### 4.1 创建知识库
1. 在扣子平台，点击"知识库"
2. 点击"创建知识库"
3. 命名为"餐饮知识库"

### 4.2 上传文档
1. 点击"上传文档"
2. 上传以下文件：
   - `knowledge-base/products.json` - 菜品信息
   - `knowledge-base/faq.json` - 常见问题

### 4.3 配置知识库
- **分段方式**：自动分段
- **索引方式**：向量索引
- **更新频率**：每天

---

## 五、测试与调试

### 5.1 测试工作流
1. 在工作流编辑器中，点击"测试"
2. 输入测试用例，查看各节点输出
3. 检查：
   - 意图识别是否准确
   - 知识库检索是否相关
   - 回复是否符合人设

### 5.2 测试用例

#### 测试用例1：打招呼
```
输入：你好
预期：热情欢迎，引导点餐
```

#### 测试用例2：菜品推荐
```
输入：有什么推荐的吗？
预期：推荐2-3个菜品，带幽默描述
```

#### 测试用例3：投诉处理
```
输入：菜太咸了！
预期：真诚道歉+幽默化解+解决方案
```

#### 测试用例4：边界测试
```
输入：你们店有大象肉吗？
预期：幽默回应+引导正经点餐
```

### 5.3 调试技巧
- 查看各节点的输入输出
- 调整提示词优化回复
- 调整温度参数控制创意度

## 七、优化建议

### 7.1 响应速度优化
- 使用轻量模型做意图识别
- 启用知识库缓存
- 使用流式输出

### 7.2 回复质量优化
- 定期更新知识库
- 收集用户反馈
- A/B测试不同提示词

### 7.3 监控与分析
- 查看对话日志
- 分析意图识别准确率
- 监控用户满意度

## 八、常见问题

### Q1：工作流节点太多，响应慢怎么办？
A：
1. 合并相似场景的节点
2. 使用轻量模型
3. 启用缓存机制

### Q2：知识库检索不准确怎么办？
A：
1. 优化文档分段
2. 调整相似度阈值
3. 使用混合检索

### Q3：回复不够幽默怎么办？
A：
1. 在提示词中增加幽默示例
2. 提高温度参数
3. 使用Few-shot提示
