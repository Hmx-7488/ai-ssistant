## 🔧 技术选型

### LLM 模型
```
选择：通义千问 qwen-plus
原因：
  - 国内访问稳定
  - 性价比高
  - 支持 Function Calling
备选：智谱 GLM-4、百度文心一言
```

### 向量数据库
```
选择：ChromaDB
原因：
  - 轻量级，无需额外部署
  - 适合开发和演示
  - LangChain 原生支持
备选：FAISS、Milvus
```

### 关系数据库
```
选择：SQLite
原因：
  - 零配置
  - 适合演示
  - 足够展示 SQL Agent 能力
备选：PostgreSQL（生产环境）
```

### API 框架
```
选择：FastAPI
原因：
  - 异步支持好
  - 自动生成 API 文档
  - 性能优秀
备选：Flask
```

## 📊 风险评估

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| LLM 幻觉 | 回答不准确 | RAG 检索 + Prompt 约束 |
| API 限流 | 响应慢 | 添加重试机制 |
| 数据不足 | 演示效果差 | 准备充足的测试数据 |
| 时间紧张 | 功能不完整 | 优先核心功能，加分项可选 |
