# 餐饮AI客服项目优化大纲

---

## 📋 优化目标

1. **意图识别准确率提升到90%+**
2. **回答与问题相关性100%**
3. **响应时间控制在2秒内**

---

## 🎯 阶段一：工作流架构修复（优先级：高）

### 任务1.1：修复节点连接关系
**预计时间**：30分钟

**步骤**：
```
1. 打开扣子工作流编辑器
2. 删除所有现有连接线
3. 重新按以下顺序连接：
   - 开始节点 → 意图识别节点
   - 意图识别节点 → 条件分支节点
   - 条件分支节点 → 知识检索节点（针对需要知识的意图）
   - 条件分支节点 → 直接服务节点（针对不需要知识的意图，如闲聊）
   - 知识检索节点 → 对应服务节点
   - 所有服务节点 → 结束节点
```

**验证方法**：
- 测试"红烧肉多少钱？" → 应该路由到价格查询
- 测试"你好" → 应该路由到闲聊

---

### 任务1.2：优化条件分支逻辑
**预计时间**：20分钟

**步骤**：
```
1. 点击"条件分支"节点
2. 设置以下分支（按优先级排序）：
   分支1: intent == "价格查询" → knowledge_retrieval
   分支2: intent == "点餐咨询" → knowledge_retrieval
   分支3: intent == "菜品推荐" → knowledge_retrieval
   分支4: intent == "投诉建议" → knowledge_retrieval
   分支5: intent == "订座服务" → knowledge_retrieval
   分支6: intent == "订单查询" → order_status_service
   分支7: intent == "闲聊互动" → general_chat
   默认: else → general_chat
```

## 🎯 阶段二：意图识别优化（优先级：高）

### 任务2.1：更新意图识别Prompt
**预计时间**：15分钟

**步骤**：
```
1. 点击"意图识别"节点
2. 清空原有prompt
3. 粘贴以下内容：
```

**Prompt内容**：
```
你是餐饮客服意图识别专家。快速判断用户意图。

用户输入: {{user_input}}

【意图分类规则】
1. 价格查询: 询问价格、多少钱、人均消费、收费、费用
   例: "红烧肉多少钱" "人均多少" "这个贵吗"
2. 菜品推荐: 想要推荐、不知道吃什么、有什么好吃的
   例: "有什么推荐" "不知道吃什么" "来点特色菜"
3. 点餐咨询: 想点餐、询问菜品详情、食材、口味
   例: "我想吃鱼" "这个菜辣吗" "有什么忌口"
4. 订座服务: 订座、预约、包间、位置
   例: "我想订座" "有包间吗" "预约明天"
5. 投诉建议: 不满、投诉、建议、问题反馈
   例: "菜太咸了" "服务态度差" "建议改进"
6. 订单查询: 查询订单、催单、上菜时间
   例: "我的菜好了吗" "等多久了" "催一下"
7. 闲聊互动: 打招呼、闲聊、其他非餐饮问题
   例: "你好" "今天心情不好" "你是谁"

请只输出意图类别名称，不要解释。
```

**设置参数**：
- temperature: 0.1
- max_tokens: 100

{{retrieved_info}}
